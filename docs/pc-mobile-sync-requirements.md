# PC・スマホ同期 要件定義書

## 1. 目的

ネットワークスペシャリスト学習アプリに、クラウドを使わずにPCとスマホの学習データを同期できる機能を追加する。

ユーザはPCとスマホの両方で日常的に学習するため、起動のたびに同期する前提にはしない。数日分または一定量まとめて学習した後に同期しても、双方の学習結果が失われず、PC・スマホの両方に反映されることを目的とする。

この機能は「バックアップ復元」ではなく「データ合流」である。読み込んだデータで現在のデータを丸ごと上書きしてはならない。

## 2. 機能名

画面上の機能名は以下とする。

```text
PC・スマホ同期
```

## 3. 基本方針

- クラウド同期は使わない。
- URL経由の同期は使わない。
- アプリ内の専用画面で同期データを生成・読み込みする。
- 同期データの受け渡し方法は、QRコードと同期文字列の両方に対応する。
- スマホからPCへ渡す場合も、PCがカメラ付きノートパソコンである可能性を考慮し、QRコード読み取りに対応する。
- カメラが使えない場合に備えて、同期文字列のコピー・貼り付けでも同期できるようにする。
- 同期は上書きではなく、未取り込みデータを追加・統合する方式とする。

## 4. 想定する利用フロー

### 4.1 スマホからPCへ同期し、PCからスマホへ返却する基本フロー

1. ユーザがスマホで学習する。
2. ユーザがPCでも学習する。
3. スマホの「PC・スマホ同期」画面で、同期データを生成する。
4. スマホ画面にQRコードと同期文字列を表示する。
5. PCの「PC・スマホ同期」画面で、スマホの同期データを読み込む。
   - PCのカメラでQRコードを読み取る。
   - または、同期文字列を貼り付ける。
6. PC側で、スマホ側の未取り込みデータをPC側データに統合する。
7. PC側で、スマホへ返す同期データを生成する。
8. PC画面に返却用QRコードと返却用同期文字列を表示する。
9. スマホの「PC・スマホ同期」画面で、PCの返却データを読み込む。
   - スマホのカメラでQRコードを読み取る。
   - または、同期文字列を貼り付ける。
10. スマホ側で、PC側の未取り込みデータをスマホ側データに統合する。
11. PCとスマホの双方で、学習状態が合流した状態になる。

### 4.2 具体例

ある日に以下の学習を行った場合：

- スマホ
  - ノートモード3件
  - 4択問題10問正解
  - 経験値Xを獲得
  - 勲章Aを獲得
- PC
  - 記述モード5問正解
  - 午後問題2問を記録
  - 経験値Yを獲得
  - 勲章B、Cを獲得

翌日に同期した後は、スマホ・PCの双方が以下の状態になること。

- ノートモード3件が反映済み
- 4択問題10問正解が反映済み
- 記述モード5問正解が反映済み
- 午後問題2問の点数記録が反映済み
- 経験値X + Yを獲得済み
- 勲章A、B、Cを獲得済み

## 5. 同期対象

### 5.1 同期するデータ

以下のデータは同期対象とする。

- クイズ進捗
- 問題ごとの習熟状態
- 問題ごとのブックマーク
- ノート理解度
- 午後問題の点数記録
- 午後問題の次回計画日
- 経験値
- 勲章
- 学習履歴
- 活動ログ

### 5.2 同期しないデータ

以下のデータは同期対象外とする。

- 午後問題の保存した解答本文
- 午後問題の一時的な入力中解答
- ログイン状態
- サイドバー開閉状態
- PWAインストール通知の非表示状態
- その他、画面表示や端末固有のUI設定

現行実装上、以下のようなキーに該当するデータは同期対象外とする。

```text
nwsp:auth
nwsp:myAnswer:*
nwsp:savedAnswers:*
```

## 6. 同期データ形式

### 6.1 同期文字列

同期文字列はURLではなく、アプリ専用の文字列とする。

形式例：

```text
NWSP-SYNC-v1:<payload>
```

`<payload>` はJSONを圧縮し、Base64URL形式などQRコードに載せやすい文字列へ変換したものとする。

### 6.2 ペイロード構造

同期データは以下のような構造を持つ。

```json
{
  "schemaVersion": 1,
  "app": "nwsp-learning-app",
  "createdAt": "2026-05-01T10:00:00+09:00",
  "fromDeviceId": "phone-xxxxxxxx",
  "sequence": 123,
  "summary": {
    "eventCount": 42,
    "xpTotalInPayload": 1240,
    "afternoonRecordCount": 2,
    "badgeCount": 1
  },
  "events": [],
  "stateFragments": {},
  "checksum": "..."
}
```

### 6.3 deviceId

各端末には永続的な `deviceId` を発行する。

- 初回起動時または同期機能の初回利用時に生成する。
- `localStorage` に保存する。
- 端末ごとに異なる値にする。
- 同期イベントIDの生成に使用する。

例：

```text
phone-7f3c2a9e
pc-4ab91e2d
```

## 7. ロールバック防止設計

### 7.1 禁止事項

同期処理で以下を行ってはならない。

```text
現在のlocalStorageを、読み込んだ同期データで丸ごと置き換える
```

この実装にすると、PCで進めた学習内容がスマホデータで消える、またはスマホで進めた学習内容がPCデータで消える可能性がある。

### 7.2 基本ルール

同期処理は以下の考え方で行う。

```text
現在の端末にあるデータ
+
読み込んだ同期データのうち、まだ取り込んでいないデータ
=
統合後データ
```

### 7.3 イベントID

各学習操作または同期可能な更新には、一意のIDを持たせる。

形式例：

```text
<deviceId>:<sequence>:<random>
```

例：

```text
phone-7f3c2a9e:00000123:ab12
```

同期時は、取り込み済みのイベントIDを記録し、同じイベントを二重に取り込まない。

### 7.4 経験値

経験値は単純な現在値の上書きではなく、経験値獲得イベントを合流して計算する。

要件：

- スマホで獲得したXPとPCで獲得したXPを合算する。
- 同じイベントIDのXPは二重加算しない。
- 可能であれば、活動ログまたはXPイベントから累計XPを再計算できるようにする。

### 7.5 勲章

勲章は和集合で統合する。

例：

```text
スマホ: A
PC: B, C
同期後: A, B, C
```

同じ勲章のXPボーナスを二重に加算してはならない。

## 8. データ別マージルール

### 8.1 クイズ進捗

- 問題IDと解答モード単位で統合する。
- 4択モードと記述モードは別々に扱う。
- 回答数・正解数は、イベントとして重複排除した上で合算する。
- 正解済み問題IDは和集合にする。
- 記述正解済み問題IDも和集合にする。

### 8.2 問題ごとの習熟状態

習熟状態は以下の順で進んだ状態とみなす。

```text
未着手 < 不正解 < 1回正解 < 連続正解
```

同期時は、より進んだ状態を採用する。

### 8.3 ノート理解度

ノート理解度は、以下の単位で扱う。

```text
categoryId + sectionIndex
```

同じ項目が両端末で更新されていた場合は、`updatedAt` が新しい方を採用する。

### 8.4 午後問題の点数記録

- 点数記録は `recordId` 単位で扱う。
- 片方にしか存在しない記録は追加する。
- 同じ `recordId` がある場合は重複追加しない。
- 午後問題の保存した解答本文は同期しない。

### 8.5 午後問題の次回計画日

次回計画日は以下の単位で扱う。

```text
problemId
```

同じ午後問題の計画日が両端末で更新されていた場合は、`updatedAt` が新しい方を採用する。

### 8.6 学習履歴・活動ログ

- 活動ログはイベントID単位で統合する。
- 片方にしか存在しないイベントは追加する。
- 同じイベントIDは重複追加しない。
- 表示時は日時順に並べる。

### 8.7 ブックマーク

問題ブックマークは和集合を基本とする。

ただし、将来的に「ブックマーク解除」も同期対象にする場合は、追加・削除をイベントとして扱い、`updatedAt` が新しい操作を採用する。

## 9. QRコード要件

### 9.1 QRコード生成

- 同期データ生成時にQRコードを表示する。
- 同じ内容の同期文字列も表示する。
- 同期文字列をコピーするボタンを用意する。
- QRコードはスマホ・PCどちらでも生成できる。

### 9.2 QRコード読み取り

- スマホ・PCどちらでもQRコード読み取りに対応する。
- カメラ権限が必要な場合は、ユーザ操作後に権限要求を行う。
- カメラが使えない場合は、同期文字列の貼り付けで代替できるようにする。

### 9.3 QRコード容量

QRコード1つで完結することを目標とする。

そのため、同期対象から以下を除外する。

- 午後問題の保存した解答本文
- 午後問題の一時的な入力中解答

同期データが大きくなりすぎる場合は、以下の順で対応する。

1. JSONを圧縮する。
2. 同期済みイベントを差分対象から外す。
3. それでもQRコード1つに収まらない場合のみ、同期文字列での受け渡しを案内する。

複数QRへの分割は、初期実装では必須としない。

## 10. 画面要件

### 10.1 入口

設定画面またはメニューに以下の導線を追加する。

```text
PC・スマホ同期
```

### 10.2 同期画面の主要操作

同期画面には以下の操作を用意する。

```text
この端末の同期データを作成
相手の同期データを読み込む
QRコードを読み取る
同期文字列を貼り付ける
```

### 10.3 同期データ生成後の表示

同期データ生成後は以下を表示する。

- QRコード
- 同期文字列
- 同期文字列をコピーするボタン
- 同期データの概要

概要例：

```text
同期データを作成しました
作成日時：2026/05/01 10:00
含まれる学習イベント：42件
含まれる午後問題記録：2件
含まれるXP：1,240 XP
```

### 10.4 読み込み前プレビュー

同期データを読み込んだ直後、統合前に内容をプレビューする。

表示例：

```text
読み込む同期データ
作成端末：phone-7f3c2a9e
作成日時：2026/05/01 10:00
追加される学習イベント：35件
追加される午後問題記録：2件
追加されるXP：1,240 XP
追加される勲章：1個
```

操作：

```text
同期する
キャンセル
```

### 10.5 同期完了表示

同期完了後は、何が追加・更新されたかを表示する。

表示例：

```text
同期が完了しました
追加された学習イベント：35件
追加された午後問題記録：2件
更新されたノート理解度：3件
追加XP：1,240 XP
追加された勲章：2個
```

PC側でスマホからの同期データを読み込んだ後は、スマホに返却するための同期データ生成を促す。

表示例：

```text
このPCにはスマホの学習データが統合されました。
スマホにもPCの学習データを反映するには、返却用QRコードをスマホで読み込んでください。
```

ボタン：

```text
スマホへ返す同期QRを作成
```

## 11. エラー処理

### 11.1 不正な同期文字列

同期文字列の形式が不正な場合は、以下を表示する。

```text
同期データを読み込めませんでした。
文字列が正しいか確認してください。
```

### 11.2 バージョン不一致

対応していない `schemaVersion` の場合は、以下を表示する。

```text
この同期データは現在のアプリでは読み込めません。
アプリを最新版に更新してください。
```

### 11.3 チェックサム不一致

データ破損が疑われる場合は、同期を実行しない。

表示例：

```text
同期データが破損している可能性があります。
もう一度同期データを作成してください。
```

### 11.4 カメラ利用不可

カメラが使えない場合は、同期文字列の貼り付けを案内する。

表示例：

```text
カメラを使用できません。
同期文字列を貼り付けて読み込んでください。
```

## 12. セキュリティ・プライバシー要件

- 同期データにはログイン情報を含めない。
- 同期データにはパスワードや認証情報を含めない。
- 同期データには午後問題の保存した解答本文を含めない。
- 同期データはユーザが明示的に生成した場合のみ表示する。
- 同期データ読み込み前に、取り込む内容の概要を表示する。
- 同期データはクラウドや外部サーバに送信しない。

## 13. 受け入れ条件

### 13.1 双方向合流

スマホとPCで別々に学習した後、スマホからPCへ同期し、PCからスマホへ返却同期を行うと、双方に両端末の学習結果が反映されること。

### 13.2 ロールバック防止

片方の端末で進めた学習内容が、もう片方の同期データを読み込むことで消えないこと。

### 13.3 XP合算

スマホで獲得したXPとPCで獲得したXPが、同期後に双方で合算されること。

### 13.4 勲章統合

スマホで獲得した勲章とPCで獲得した勲章が、同期後に双方で獲得済みになること。

### 13.5 重複取り込み防止

同じ同期データを複数回読み込んでも、XP・学習履歴・午後問題記録・勲章が二重に加算されないこと。

### 13.6 午後問題の保存解答本文は同期しない

午後問題の点数記録は同期されるが、保存した解答本文は同期されないこと。

### 13.7 QRと文字列の両対応

スマホ・PCともに、QRコード読み取りと同期文字列貼り付けの両方で同期できること。

### 13.8 QR 1つでの返却同期

午後問題の保存解答本文を除外した通常利用の差分同期において、PCからスマホへの返却同期はQRコード1つで完結できることを目標とする。

## 14. 初期実装で必須ではないもの

以下は初期実装では必須としない。

- クラウド同期
- アカウント単位のサーバ保存
- 複数QRコード分割
- リアルタイム同期
- 端末間の自動検出
- 午後問題の保存解答本文の同期
