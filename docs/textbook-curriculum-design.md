# 教科書モード カリキュラム詳細設計（全体マップ）v1

作成日: 2026-06-23

本書は `docs/textbook-mode-requirements.md`／`docs/textbook-mode-basic-design-v2.md` を上位に置く「詳細設計（カリキュラム層）」である。第1章の実装・レビューで固めたトーン／図解／スマホ表示の型を、第2〜20章へ展開するための **全体の流れ** を設計する。

> この版は **マップ（章順・初出概念・前提・構成図の変化・午後フック）** までを対象とする。各章の節レベル詳細設計（本文ドラフト・図のステップ）は、本マップ合意後に章ごとに作成する。

---

## 0. 設計の背骨（これが「全体の流れ」）

**1つのネットワーク構成図が、章を追うごとに育っていく。** 第1章の最小構成（PC—L2SW—ルータ—Webサーバ）から始め、毎章「現実の要素」を1つずつ足し、最終章ではネスペ午後相当の全体図になる。学習者は同じ図が育つのを見続けることで、午後の構成図に**見慣れる**。

各章は次の2軸で前進する。

- **縦の深掘り（ズームイン）**: ある通信や機器の中身を詳しく見る（構成図は増やさず、注目領域を拡大）。
- **横の拡張（構成図を育てる）**: 機器・ゾーン・経路・冗長を1つ足す（構成図そのものが成長）。

この2軸を交互に使い、「詰め込みすぎず・1図1理解単位・スマホ1画面」を保つ。

## 2. 章マップ（4部構成）

各章 = 「この章で初出させる概念」「前提（既習）」「構成図の変化」「ネスペ午後フック」。

### 第1部　まず1つの通信を最後まで（最小構成のままズームイン）
第1章の「PC→Webの旅」を完結させる。新しい機器は増やさず、旅の各段階を深掘り。

- **1. OSI参照モデルと通信の全体像【済】**
  初出: OSI／カプセル化／MAC・IP・ポート／ARP／デフォルトGW／同一・別ネットワーク判定／通して見る。午後: 境界・次ホップ・層での切り分けの素地。

- **2. 名前解決とアドレス配布（DNS・DHCP）**
  初出: DNSで名前→IP、DHCPで自分のIP・GW・DNSをもらう仕組み。前提: 1章のIP/GW。図: 内部LANにDNS/DHCP追加。午後: 名前解決の失敗切り分け、DHCPスコープ。
  *ねらい: 1章は「Webサーバの IP を知っている前提」で進めた。その IP をどう得るか＝旅の出発前を埋める。*

- **3. TCP・UDPとポート番号**
  初出: TCP3wayハンドシェイク／UDP／ポート／5タプル／コネクション・再送。前提: 1章のL4ポート。図: 同じ構成で「会話」をズーム。午後: 5タプルで通信を識別、FW許可条件の素地。

- **4. Web通信の中身（TLS・HTTP）**
  初出: TLSハンドシェイク／サーバ証明書／HTTPの要求応答。前提: 3章TCP。図: 443の中身をズーム。午後: HTTPS通信の構成、証明書まわり。

### 第2部　ネットワークの土台を組む（構成図を育てる：機器とアドレス）
旅の舞台＝ネットワーク自体の作りと、アドレス／経路の設計を学ぶ。構成図が横に育ち始める。

- **5. L2スイッチング・VLAN・STP**
  初出: MAC学習／VLAN／タグVLAN／ブロードキャストドメイン／STPでループ防止。前提: 1章L2/MAC。図: 内部LANが複数SW＋VLAN＋冗長リンクへ。午後: VLAN設計、ループ。

- **6. IPアドレス設計・サブネットとルーティング**
  初出: サブネット計算／CIDR／経路表／ロンゲストマッチ／スタティック→OSPF。前提: 1章のネットワーク判定（**ここで正式化**）。図: 複数セグメント＋2台目ルータ。午後: 経路・次ホップ・サブネット（午後頻出）。

- **7. インターネット接続・NAT・BGP**
  初出: グローバル/プライベート／NAPT／AS／BGPで外部到達。前提: 6章ルーティング。図: インターネット境界を追加。午後: NAT変換、外部到達性。

### 第3部　現実の構成へ（境界・拡張・冗長：午後構成図に近づける）
企業ネットの実物に近づける。ここから構成図はネスペ午後スケールへ。分量が大きいので2クラスタで捉える：**8〜11章＝境界と拡張**／**12〜18章＝領域別**。

- **8. セキュリティ境界・ファイアウォール・DMZ**
  初出: FW／許可条件／ステートフルインスペクション／DMZ／三層境界。前提: 3章ポート、7章境界。図: 内部/DMZ/外部の三層化、FW挿入。午後: **境界と許可条件の読解（午後最重要）**。

- **9. ロードバランサ・プロキシ・CDN**
  初出: L4/L7 LB／リバース・フォワードプロキシ／CDN。前提: 4章HTTP、8章境界。図: サーバゾーンにLB＋サーバ群、出口プロキシ、外部CDN。午後: 振り分けと経路。

- **10. 冗長化・可用性（と無停止更改）**
  初出: 機器/経路/リンク冗長、VRRP／リンクアグリゲーション、フェイルオーバー、更改の段取り。前提: 5〜9章。図: 境界/ルータ/LBを冗長ペアに。午後: 単一障害点、切替。

- **11. 拠点間接続・VPN・WAN**
  初出: IPsec／拠点間VPN／WAN／SD-WAN／オーバーレイ。前提: 7章境界、8章FW。図: 2拠点目を追加しVPN/WANで接続。午後: 拠点間経路、トンネル。

- **12. 認証・認可・SSO・PKI**
  初出: 認証と認可／RADIUS／SSO／証明書チェーン／PKI。前提: 4章TLS証明書。図: 認証サーバ(RADIUS)をオーバーレイ。午後: 認証フロー。

- **13. 無線LAN**
  初出: AP／WLC／SSID／IEEE802.1X認証／ローミング。前提: 5章VLAN、12章認証。図: 内部LANにAP/WLC。午後: 無線設計。

- **14. IPv6**
  初出: IPv6アドレス／NDP／SLAAC／デュアルスタック。前提: 6章アドレス、1章ARP↔NDP対比。図: 既存にIPv6を再アドレス。午後: IPv6移行。

- **15. メール**
  初出: SMTP配送経路／SPF・DKIM・DMARC。前提: 2章DNS、8章DMZ。図: DMZにメールサーバ、外部MX。午後: 送信ドメイン認証。

- **16. 仮想化・クラウド（SDN/VXLAN/EVPN）**
  初出: サーバ仮想化／VPC／SDN／VXLAN／EVPN。前提: 5章VLAN、7章。図: クラウドゾーン(VPC)＋オーバーレイ。午後: クラウド接続。

- **17. VoIP・QoS・マルチキャスト**
  初出: VoIP／QoS優先制御／マルチキャスト。前提: 3章UDP、5/6章。図: IP電話＋QoSマーキング。午後: QoS設計。

- **18. IoT・LPWA（概要）**
  初出: IoT通信像／LPWA省電力広域（概要のみ・深追いしない）。前提: 全般。図: IoTゲートウェイ＋デバイス（軽量）。午後: 軽め。

### 第4部　運用と総合
- **19. 運用監視・障害切り分け**
  初出: 監視（SNMP/syslog/フロー）／切り分けの段取り。前提: 全章。図: 監視サーバ追加、「どこを見るか」。午後: 障害切り分け。

- **20. 午後問題の読み方・総合演習**
  初出: なし（統合）。前提: 全章。図: これまでの積み上げ＝午後相当の全体図。午後: 構成図＋設問の総合読解。

## 3. 現行リスト（index.ts）からの主な変更と理由

| 変更 | 現行 | 提案 | 理由 |
|---|---|---|---|
| 名前解決を接続より前へ | 2=TCP, 3=DNS/DHCP/ARP, 4=Web/TLS | 2=DNS/DHCP, 3=TCP, 4=TLS/HTTP | Webを開く順序＝名前解決→TCP→TLS。旅の時系列に合わせる |
| ARPの重複排除 | 3にARP | 3からARP削除（1章で完了） | 同じ概念の二度出しを避ける |
| FW/境界を前倒し | 11=FW | 8=FW・DMZ | 境界は午後の核。LB/冗長/VPNは「境界が立った後」に乗る |
| 第3部の並べ替え | 8冗長,9LB,10VPN,11FW,12認証 | 8FW,9LB,10冗長,11VPN,12認証 | 構成図の成長順（境界→配信→冗長→拠点→認証）に整列 |

章数は20のまま。順序と各章の輪郭を、依存関係と構成図の成長で整えた。

## 4. 決定事項（合意済み）

1. **サブネット計算は第6章で正式化**（第1章は概念のみ／スパイラル）。第2〜5章は前方参照に留める（chapter-designs「用語の前方参照ルール」）。
2. **DNS/DHCPを第2章へ前倒し**（TCPより先＝旅の時系列）。
3. **FW・DMZを第8章へ前倒し**（境界を午後の核として早く立てる）。
4. **第1部は1リクエスト完結**（DNS→TCP→TLS、機器を増やさずズーム）。
