# 教科書モード 図解仕様書 v2

最終更新: 2026-07-13（第16章メールを追加・role mail・全章のダーシ撤去）

「どの概念を、どの図で、どう見せれば一番わかりやすいか」を確定する。図解は本モードの最重要・差別化要素であり、**図の選択と画面仕様は本書を正とする**（章別設計 `textbook-chapter-designs.md` の図は本書に従う）。§3 のカタログはすべて実装済み。

---

## 1. 設計原則

1. **1図＝1理解単位**。詰まるなら図を分ける／ステップに分ける。逆にスカスカで意味の薄い図は載せない。
2. **図の形は概念に合わせて選ぶ**（§2）。経路→構成図を旅する図、順序→縦のシーケンス、構造→入れ子/対比、規則→表。
3. **構成図は実トポロジで描く**。一直線に潰さない（星形・ループ・三角形・縦積みは graph レイアウト）。
4. **各図で最も分かりやすい形を優先**。共通化のために合意済みの見せ方を崩さない。
5. **動きには必ず役割**（順を追う・変化を見せる・前後比較）。装飾アニメ・無限アニメ禁止。
6. **図にメタ情報を足さない**（「追加」バッジ等は不採用実績あり）。注記・差分はステップ説明文で。
7. **必要な情報（IP・次ホップ・コスト等）は図の中に書き込む**。ただしノード sub は短く（SVGで約92pxに収まる長さ）。
8. **ノード・ラベルを覆う要素を作らない**。動きは線の強調＋方向矢印＋小さなマークで表す。
9. **スマホ最優先・1画面**。375/360で横スクロール0。操作ボタンは動かない（固定高設計）。
10. **新しい見せ方は show_widget のモック（実装と同じ座標・色）で方向合意してから実装**。

## 2. 図の使い分け（概念タイプ → 最適な図）

| 見せたいことの主役 | 図 | 例 |
|---|---|---|
| 層の地図（いまどの層か） | `osi-stack` | OSI 7層 |
| ヘッダの入れ子・包む/外す | `encap` | カプセル化、IPsec、VXLAN |
| 数項目を同じ目線で対比 | `address-table` | MAC/IP/ポート、TCP/UDP、プライベート/グローバル |
| **一直線の経路**を旅する・付け替え・書き換え | `packet-flow`（chain） | 第1章通信フロー、第8章NAPT往復 |
| **実トポロジ**（星形・ループ・三角形・多段）上の動き・地図ツアー | `packet-flow`（graph） | VLAN、STP、OSPF三角形、経路表連動、全体図の成長 |
| 2〜3者のやり取りの**順序** | `sequence` | DHCP・TCP 3ウェイ・TLS・HTTP・DNS・RADIUS |
| 規則／対応表（特定行を強調） | `record-table` | 経路表、FWルール、行き/戻り対称表 |
| アドレスの内部構造 | `subnet-calc` | ネットワーク部/ホスト部 |
| セグメントとGWの読み取り | `segment-map` | ホストIP→所属ネットワーク導出 |
| 手順の時系列俯瞰 | `timeline` | 第1部パノラマ、無停止更改、証明書チェーン、CSMA/CA |
| **電波の届く範囲**（空間の位置関係） | `radio-range` | 隠れ端末（第14章） |
| **IPv6アドレスの表記と構造** | `ipv6-address` | フル→::省略→前半/後半（第15章） |

> **packet-flow か sequence か**: 主役が経路・機器処理・ヘッダ変化なら packet-flow、2〜3者の会話の往復なら sequence（1本の線上で往復させない）。
> **chain か graph か**: 通信が一直線に流れる（第1〜4章型・NAT越え）なら chain。スイッチに端末がぶら下がる／冗長ループ／3ルータ相互接続／ルータ多段の全体図なら graph。

## 3. 図カタログ（確定仕様・全て実装済み）

各図: 目的 / 画面構成（モバイル基準）/ 動き / Do・Don't。データ型は `src/data/textbook/types.ts` が実体。

### 3.1 `osi-stack`
- 7層を縦に。主役層（L1〜L4）は色、上位は淡色。静的（任意で段階ハイライト）。
- Do: 主役を強調・上位をdim。 Don't: 7層を同じ濃さで詰める。

### 3.2 `encap`
- **「前→後」比較**（第12章レビューで確定）: 各ステップで包む/外す前（前）と後（後）を、wide=左右／narrow=上下に並べ、増えた／外れた層を枠＋バッジ（＋増えた／外す）で強調。「どう変化したか」を見比べられる（単一状態の逐次表示は分かりにくいと差し戻し）。
- `levels` は**任意段数**（外側→内側。ラベル・トレーラ・色を外から渡す）。IPsec（第12章: 新IP→IPsec→元IP）・VXLAN（第17章）もこの形。入れ子は常に横方向で背を低く保ち、ステージ高さは幅で切替（narrow=高め・wide=低め）＝各幅でボタン不動。3段以上で使う（2層は余白過多、第4章で削除実績）。

### 3.3 `address-table`
- 対比カード（モバイル=1カラム縦積み、PC=横並び）。カード=見出し＋layerチップ＋定義リスト。**PCの列数は枚数に応じて切替**（2枚=2カラム・3枚以上=3カラム。2枚を3カラムに置くと右1/3が空く＝第13章で差し戻し）。
- 定義リストの項目ラベルは既定「何を示すか／届く範囲／例」。**軸が合わない対比は `fieldLabels` で3見出しを差し替える**（第13章 認証vs認可=「答える問い／判断のよりどころ／例」。第15章 IPv4vs6 でも利用想定）。カードの型（見た目）は変えない。「例」の等幅フォントは既定軸（アドレス例示）のみで、fieldLabels 指定時は通常フォント（日本語の例文）。表形式が合うものは `record-table`（rowHeader）。

### 3.4 `packet-flow` 共通仕様
- `steps[]` の `focus`（link/node）で現在区間・処理を指す。`explanation` は固定高（約2行・全角換算50字目安）。
- **ヘッダ表**（L2/L3/L4 の現在値＋「変わる/そのまま/外す」チップ）は付け替え・書き換えが主題の図で使う。不要なら `hideHeaders`。
- **sideTable**（ステップ連動で埋まる表）: 経路表・MAC表・NAPT変換表に。**行数は全ステップで固定**（未記録は「—」）にしてボタン不動を保つ。`tableHighlightRow` で現在行を強調。ヘッダ表と sideTable の併用はしない（縦に重すぎる）。
- 拡張（すべて実装済み）: `verdict?: 'pass'|'block'`（第9章FW・ノード脇へ通過/遮断チップ＝ノードは覆わない）／`bubbles`（宛先・送信元の吹き出し。graph=左脇固定／chain=回線中央／tunnel=帯の下の箱に1〜2行）／`downNodes`（第10章LBヘルスチェック・第11章フェイルオーバー。停止ノードを灰色表示。枝の`blockedLink`✕と併用可）／`pairActive`（第11章VRRP・pairモードで稼働中のノード。他方は待機中、`downNodes`は故障として状態チップに反映）。

### 3.5 chain レイアウト（`TopologyView`・既定）
- 座標レスの一直線。ノードは zone ごとに破線ボックスで囲む（zone ラベルに CIDR を含めてよい: 「内部LAN 192.168.10.0/24」）。
- PC=横並び。**ゾーンボックスの伸長率はノード数比**（1ノード箱は伸ばさない）とし、余白を回線に配分して吹き出しの居場所を確保。スマホ=全幅カードの縦リフロー。
- **パケット表示**: 線上を動くのは**小さな封筒マークだけ**。ラベルは**回線中央に固定した吹き出し**（PC=線の上・下向き三角、スマホ=縦線の右・左向き三角）。「送信元/宛先 ＋ 値」形式のラベルは自動で**2行組**になり幅を抑える。パケットラベルは図中表記なので「宛先」（§4）。
- Don't: ノードを覆う長いチップ（差し戻し実績）。吹き出しとノードアイコンの重なりは検証で0を確認する。

### 3.6 graph レイアウト（`GraphTopology`・SVG 320幅）
`Topology.layout:'graph'`。スイッチ/ルータ/FW/雲=幹（spine）、PC・サーバ等=枝（leaf）。動きは**アクティブな線＋進行方向の矢印**（チップは出さない。ただしFWの判定は `verdict` チップを脇に）。`Topology.leafIds` でロールに関わらず葉に格下げできる（例: BGP図で境界ルータを雲の枝に）。8モード（pair/bundle/tunnel は spine もロール色で塗る＝既存5モードの spine=slate と異なる）:

| モード | 判定 | 用途・要点 |
|---|---|---|
| **tree** | 既定 | スイッチ横一列＋端末を下に縦積み（第5章VLAN、第2章の構成図）。幹どうしの線のラベルは既定「トランク」、`edgeLabels` で上書き（`''`で非表示） |
| **loop** | 同一2ノード間に links 2本 | スイッチ縦並び＋曲線アーク2本（第5章STP）。`PacketStep.blockedLink` で片アーク破線＋✕ |
| **triangle** | spine 3台が相互リンク | ルータ/AS 3つの三角形（第7章OSPF、第8章BGP）。`edgeLabels[{a,b,label}]` でコスト/帯域/BGPラベル、`blockedLink` で切断→迂回 |
| **stack N段** | `stack:true`・spine 2台以上 | spine（nodes配列順=上→下）を縦一列。最上段の葉=上の横並び・最下段=下・**中間段=左右の短い水平枝（最大2）**。最上段に葉が無ければ余白を詰める。全ノード1枚の全体図・経路表連動に（第7章§2=2段、第8章§1=4段） |
| **tiers（三方向FW）** | `tiers:true`・spine 2台以上 | spine を縦一列（上=外部・下=内部）＋**最下段のノード（FW）から zone ごとに左右の列へ枝分かれ**（列は zone 初出順に左→右。列内は縦積み）。第9章 内部/DMZ/外部の三層境界。枝は `blockedLink` で赤破線＋✕、`PacketStep.verdict` でFW脇に通過/遮断チップ |
| **pair（冗長ペア）** | `pair:true`・spine 3台以上 | 上=共有先（インターネット等）を中央、中段に**2台の冗長ペアを左右**、下=端末（PC）を中央の3段。仮想IP `vip` を**ペアと端末の間に固定表示**（PCのGW＝この仮想IP）。稼働=`pairActive`／待機=他方／故障=`downNodes` を、各ペア下の**状態チップ**（稼働中/待機中/故障）で表示。チップ文言は `pairChipLabels` で差し替え可・standby `''` で非稼働側を非表示（第14章ローミング=「接続中」だけがAP1→AP2へ移る。両APとも稼働しているため「待機中」は誤り）。故障ノードの枝は自動で赤破線＋✕。第11章VRRP・第14章ローミング |
| **bundle（LAG）** | `bundle:true`・spine 2台に links 2本以上 | 2台を左右に置き、間の複数リンクを**近接した平行線＋点線ブラケット**で1本の論理リンクに（`bundleNote` をブラケット上に）。`blockedLink` がそのペアに一致で**上側リンクに✕**（1本故障＝残りで継続）。帯域は `bundleBandwidth{full,reduced}` を下段に表示し blocked で自動的に reduced へ。第11章LAG |
| **tunnel（拠点間VPN）** | `tunnel:true`・spine 2台 | 2台の拠点ルータを横に、間を**暗号トンネルの帯**（`tunnelNote` を帯上に）で結ぶ。端末は各ルータの真下（leaf）。focus が 2ルータ間リンクのとき帯を強調。二重IPは `PacketStep.bubbles` を帯の下の箱に1〜2行で出す（段階的に外側/中身が増減＝折り返さないSVGで常に1つながり）。第12章IPsec VPN |

- ゾーンは葉の色分け＋白チップのセグメント名ラベル（短く。CIDRは入れない）。
- **無線リンクは破線**: `Topology.wirelessLeafIds`（無線接続の葉id）で、その葉へ入る枝を細い破線にして有線と区別（第14章 端末—AP間。遮断の太い赤破線とは別物）。role `ap` は spine 扱い（端末がぶら下がる幹）。WLC も role=ap だが `leafIds` で葉に落とす。
- **パケットの宛先/送信元**は `PacketStep.bubbles`（「宛先 X」「送信元 Y」形式・最大2）で、アクティブ対象の**左脇に固定**表示（chain の吹き出しと同形＝白地＋青枠・2行組・三角で対象を指す）。中央縦spineのノード列に食い込ませない＝ノード非被覆。**右脇は verdict チップ専用**で左右に棲み分け（tiers 等の中央縦spine向け。第9章）。
- Do: 検証で全 `<text>` が viewBox 内にあること。 Don't: 実トポロジを chain の一直線に潰す。

### 3.7 `sequence`
- 2〜3者のラダー。全メッセージを最初から描画し現在だけ強調＝**構造的にボタン不動**。ブロードキャストは破線。
- アクターはモバイル最大3。4者以上は「中心1者＋相手を段階ラベルで分解」——DNS は cache→上位DNS への矢印を「②ルートへ ③TLDへ ④権威へ」と**3本に分けて反復を可視化**する（第2章で確立）。

### 3.8 `record-table`
- 行=1レコード。スマホ=1行1カード（列ラベル:値の縦並び）、PC=表。`highlightRow`・`rowHeader`・`emphasizeKey`。
- モバイルの列は3〜4に絞る。セルも文体規則の対象（常体禁止・図中は「宛先」）。

### 3.9 `subnet-calc`
- 4オクテットを10進＋2進で縦2段、プレフィックス境界で緑/グレー。steps で /24→/26 と境界が動く。

### 3.10 `segment-map`
- ルータが2セグメントをつなぐ固定構図。ホストIPからネットワークアドレスを段階的に書き込み、ルータ両端IP（GW）も図中に表示（第6章）。

### 3.11 `timeline`
- 縦タイムライン（バッジ＋短文）。全項目描画＝ボタン不動。第1部パノラマ・無停止更改・証明書チェーンに。
- **「自分の動作」は sequence にしない**: キャリアセンス（自分が聞く）のような矢印にならない手順は timeline で俯瞰する（第14章 CSMA/CA で確立）。

### 3.12 `radio-range`（第14章で新設）
- **電波の届く範囲を円で描く固定構図**: 左右の端末＋上中央AP。円＝その端末の電波の届く範囲（縁は破線＝無線の文法）。どちらの円にもAPは入るが、端末どうしは互いの円の外＝**隠れ端末の位置関係**。
- steps: `show:'left'|'both'` で円を段階表示、`collide:true` で同時送信の2本線＋APに✕。SVG 320×214・説明文 h-12・ボタン不動。
- 円の外周が図の端で少し切れるのは「電波の広がり」として許容（テキストは viewBox 内厳守）。
- Do: 空間の位置関係が本質の概念（隠れ端末・電波干渉）に。 Don't: 経路や会話の表現（packet-flow / sequence の領分）。

### 3.13 `ipv6-address`（第15章で新設）
- **16進8グループを4×2段**（上段=前半64ビット・下段=後半）で固定表示し、ステップで見え方だけを変える: `mode:'full'`（フル表記）→ `'compressed'`（先頭0を薄く・全0グループをamberで強調、下の「いまの表記」が::省略形に変化）→ `'structure'`（前半=緑・後半=グレーの色分け）。
- 配色は `subnet-calc` と同じ（緑=ネットワーク／グレー=ホスト）＝「第6章と同じ発想」を視覚でも引き継ぐ。凡例は常時表示。
- 「いまの表記」行は固定高（スマホでフル表記が2行になるため。可変にするとボタンが動く＝実測で検出した失敗）。

### 確認問題（`check` ブロック・章末 `checks`）
図ではないが視覚ユニット。質問カード＋「答えを見る」開閉。本文中は label（'設問例'）、章末は Q1〜Q3。リッチテキスト装飾可。仕様は基本設計 §5。

## 4. 構成図の共通規約

- **向き**: 境界が登場する第8章以降の全体図は**上=外部（インターネット）・下=内部**に統一。
- **アドレス**: 章をまたぐ一貫アドレス台帳（`textbook-chapter-designs.md`）に従う。台帳に無い値を図に発明しない（架空例を使うときは「架空」と明示し、実構成と混ざらないアドレス帯を使う）。
- **ゾーンの色（tone）慣例**: sky=内部LAN／emerald=別フロア／amber=サーバLAN（第9章以降DMZ）／violet=インターネット境界／slate=インターネット・社外。
- **ノード**: label は短く（graph の spine 箱は約7文字）、sub に IP や役割（graph は約92px以内）。ロールと機器名は実務語（L2SW・ルータ・境界ルータ等。造語不可）。
- **パケットラベル**: 「送信元 192.168.10.10:51000」「宛先 172.16.0.20」の形式（図中なので「宛先」）。
- **領域フォーカス（`zoneFocus`）は不使用**（凍結）: 俯瞰＋現在ゾーン詳細の2段表示は第7章で「中途半端な省略」と差し戻し。実装は残っているが、複雑な構成図は **stack N段の全ノード1枚**を第一候補にする。

## 5. 章 × 図 割り当て

### 実績（第1〜10章・公開済み）

| 章 | 図 |
|---|---|
| 1 | osi-stack / encap / address-table / packet-flow chain ×2（ARP・通信フロー） |
| 2 | packet-flow graph tree（構成図: DNS/DHCP追加） / sequence ×2（DORA・DNS 6メッセージ） / timeline |
| 3 | sequence（3ウェイ） / record-table ×3（TCP/UDP・5つの情報・代表ポート） |
| 4 | sequence ×2（TLS・HTTP） / record-table（メソッド/ステータス） / timeline（第1部パノラマ） |
| 5 | packet-flow chain＋sideTable（MAC学習） / graph tree（VLAN） / graph loop ×2（ストーム・STP） |
| 6 | subnet-calc / segment-map / record-table（VLSM・架空例 192.168.50.0/24） |
| 7 | graph stack 2段＋sideTable（経路表連動） / record-table ×4 / graph triangle（OSPF切替） |
| 8 | graph stack 4段（全体図） / address-table（私設/グローバル） / chain＋sideTable（NAPT往復・吹き出し） / graph triangle＋leafIds（BGP迂回） / record-table（行き/戻り） |
