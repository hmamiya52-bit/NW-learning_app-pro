## 1. プロジェクト概要

### 1.1 目的・背景

ネットワークスペシャリスト試験（情報処理技術者試験）の合格を目指す学習者が、個人作成の復習ノート（85ページ、PDF）の内容を効率的に学習・暗記できるWebアプリケーションを構築する。

PDFは3段階の構成（解答付き → 一部空欄 → 空欄多め）であり、この段階的な難易度構造を活かした学習体験を提供する。加えて、令和7年4月試験で出題されたが復習ノートに含まれていないトピックを補足コンテンツとして追加する。

### 1.2 ターゲットユーザー

- ネットワークスペシャリスト試験の受験者
- 応用情報技術者レベル以上の基礎知識を持つ学習者

### 1.3 配布形態

- 作成者による個人配布（認証機能は不要）

### 1.4 解決する課題

| # | 課題 | アプリによる解決 |
|---|------|-----------------|
| 1 | PDFでは自分の理解度を把握しづらい | トピックごとの正答率・進捗を可視化 |
| 2 | 暗記確認を効率的に繰り返せない | 穴埋めクイズ形式でインタラクティブに反復学習 |
| 3 | 弱点分野が特定しにくい | 正答率の低い分野を自動抽出し優先的に出題 |
| 4 | 学習の継続が難しい | 学習記録・連続学習日数で動機付け |
| 5 | 最新の試験傾向に対応できていない | 令和7年出題トピックの補足コンテンツで対応 |

## 4. データ構造

### 4.1 エンティティ定義

**Category（カテゴリ）**
- id: string -- カテゴリの一意識別子（例: "lan-basics", "firewall"）
- name: string -- 表示名（例: "レイヤ1〜3基礎"）
- order: number -- 表示順序
- description: string -- カテゴリの概要説明

**Topic（トピック）**
- id: string -- トピックの一意識別子
- categoryId: string -- 所属カテゴリのID
- name: string -- トピック名（例: "VLAN", "OSPF"）
- order: number -- カテゴリ内の表示順序
- content: string -- 詳細解説のHTMLコンテンツ（PDF前半に対応）
- source: "pdf" | "r7-exam" -- コンテンツの出典（PDF原本 or 令和7年試験補足）

**Question（問題）**
- id: string -- 問題の一意識別子
- topicId: string -- 所属トピックのID
- type: "fill-in-blank" -- 問題形式（穴埋め。解答時に4択モード/記述モードをユーザーが選択）
- questionText: string -- 問題文（空欄箇所を `{{blank}}` でマーク）
- correctAnswer: string -- 正解
- choices: string[] -- 4択モード用の選択肢（正解1つ + 誤答3つ）
- isImportant: boolean -- 重要問題フラグ
- explanation: string -- 解説文
- difficulty: 1 | 2 | 3 -- 難易度（1:基礎、2:標準、3:応用）

**AnswerRecord（解答記録）** -- ローカルストレージに保存
- id: string -- 記録の一意識別子
- questionId: string -- 解答した問題のID
- mode: "multiple-choice" | "written" -- 解答モード（4択 or 記述）
- isCorrect: boolean -- 正誤（4択: 自動判定、記述: ユーザー自己判定）
- userAnswer: string -- ユーザーの解答内容
- answeredAt: string (ISO 8601) -- 解答日時

**UserProgress（学習進捗）** -- ローカルストレージに保存
- topicId: string -- トピックID
- totalAttempts: number -- 総解答数
- correctCount: number -- 正解数
- lastStudiedAt: string (ISO 8601) -- 最終学習日時
- isBookmarked: boolean -- ブックマーク状態

**StudySession（学習セッション）** -- ローカルストレージに保存
- id: string -- セッションの一意識別子
- startedAt: string (ISO 8601) -- 開始日時
- endedAt: string (ISO 8601) | null -- 終了日時
- mode: "topic" | "weakness" | "random" | "important" -- 学習モード
- categoryId: string | null -- カテゴリ指定モードの場合のカテゴリID
- questionCount: number -- 出題数
- correctCount: number -- 正解数

**Bookmark（ブックマーク）** -- ローカルストレージに保存
- questionId: string -- ブックマークした問題のID
- createdAt: string (ISO 8601) -- ブックマーク日時

### 4.2 リレーション

```
Category —(1:N)— Topic
Topic —(1:N)— Question
Question —(1:N)— AnswerRecord
Topic —(1:1)— UserProgress
StudySession —(1:N)— AnswerRecord
Question —(0:1)— Bookmark
```

## 5. 技術構成

### 5.1 アーキテクチャ

- **形式**: SPA（シングルページアプリケーション）
- **ホスティング**: 静的サイトとしてデプロイ可能（サーバーレス）
- **データ保存**: ブラウザのLocalStorageを使用（バックエンドサーバ不要）
- **学習コンテンツ**: ビルド時にJSONとしてバンドル（PDFから手動/スクリプトで変換）

### 5.2 技術スタック（推奨）

| レイヤ | 技術 | 理由 |
|--------|------|------|
| フレームワーク | React または Vue.js | SPA構築に適している |
| スタイリング | Tailwind CSS | ユーティリティファーストで効率的なUI構築 |
| 状態管理 | ローカルステート + LocalStorage | バックエンド不要でシンプル |
| ビルドツール | Vite | 高速な開発サーバーとビルド |
| テスト | Vitest | Viteとの統合が容易 |

### 5.3 非機能要件

| # | 項目 | 要件 | 優先度 |
|---|------|------|--------|
| NF-01 | レスポンシブ対応 | PC・タブレット・スマートフォンで利用可能 | 必須 |
| NF-02 | PWA対応 | Service Workerによるオフライン学習、ホーム画面追加、キャッシュ戦略（Cache First） | 必須 |
| NF-03 | パフォーマンス | 画面遷移は200ms以内に完了 | 必須 |
| NF-04 | データ永続性 | ブラウザのLocalStorageにデータを保存、クリアしない限り維持 | 必須 |
| NF-05 | アクセシビリティ | キーボード操作でクイズに解答可能 | 推奨 |

## 6. スコープ定義

### 6.1 対象範囲（IN）

- 復習ノートPDF（85ページ）の内容に基づく学習コンテンツ
- 令和7年4月試験で出題されたが復習ノートに含まれていないトピックの補足コンテンツ
- 穴埋めクイズ・4択クイズによるインタラクティブ学習
- トピック別の進捗管理・正答率の可視化
- ブラウザ上で完結するSPA（バックエンド不要）
- LocalStorageによるデータ永続化
- レスポンシブデザイン（PC/タブレット/スマホ対応）
- PWA対応（オフライン学習、ホーム画面追加）

### 6.2 対象外（OUT）

- ユーザー認証・ログイン機能
- 複数ユーザー間のデータ共有・ランキング機能
- サーバーサイドのデータ保存・同期
- PDFからの自動コンテンツ抽出（コンテンツは手動でJSON化する前提）
- 午後試験の記述式問題対応
- 新しい学習コンテンツの動的追加（管理画面）
- ネイティブモバイルアプリ

## 7. コンテンツ変換方針

PDFから学習コンテンツへの変換は以下の方針で行う。

### 7.1 PDFの構造

PDFは同一トピックを3段階の難易度で構成している：

| 区分 | ページ | 記号 | 内容 | 用途 |
|------|--------|------|------|------|
| 解答付き（詳細版） | 1〜29 | ◆ | 全項目が記載された完全な解説 | 詳細解説ビュー / 正解の根拠 |
| 一部空欄 | 29〜53 | ◇ | 重要キーワードの一部が空欄 | 基礎レベルの穴埋めクイズ |
| 空欄多め | 53〜81 | ◇ | より多くの箇所が空欄 | 応用レベルの穴埋めクイズ |
| 追加分 | 82〜85 | - | 上記に含まれる内容（変換時に統合） | - |

### 7.2 コンテンツ変換方針

**詳細解説（ページ1〜29）**
- 各◆トピックをTopicエンティティとして構造化
- 解説文をHTML形式に変換（箇条書き、表、強調を保持）
- 試験対策のコツは特別なスタイルで強調表示

**穴埋めクイズ — 基礎レベル（ページ29〜53: 一部空欄）**
- 空欄箇所をQuestion（fill-in-blank, difficulty: 1）として抽出
- 空欄が少ないため、基本的な用語・概念の確認に適する
- 各問題に4択用の選択肢（正解1 + 誤答3）を付与

**穴埋めクイズ — 応用レベル（ページ53〜81: 空欄多め）**
- 空欄箇所をQuestion（fill-in-blank, difficulty: 2〜3）として抽出
- 空欄が多いため、より深い理解を問う問題となる
- 各問題に4択用の選択肢（正解1 + 誤答3）を付与

※ 全ての穴埋め問題は、ユーザーが解答時に「4択モード」「記述モード」を選択できる

### 7.3 重要問題の判定基準

以下の条件に該当する問題には `isImportant: true` を設定する。

| 判定元 | 条件 | 理由 |
|--------|------|------|
| ページ1〜29（詳細版） | **太字の赤字**で記載されている箇所から作成した問題 | 著者が特に重要と判断した知識 |
| ページ29〜53（一部空欄） | **空欄になっている箇所**から作成した問題 | 著者が最優先で暗記すべきと判断した知識（空欄が厳選されているため） |

- ページ53〜81（空欄多め）の空欄は網羅的であるため、重要フラグの対象外とする
- UIでは重要問題に視覚的なマーク（例: 星マーク）を表示する

**追加分（ページ82〜85）**
- 内容は前半に含まれるため、変換時に該当トピックに統合する

### 7.4 令和7年試験 補足コンテンツ

復習ノートに含まれていないが令和7年4月試験で出題されたトピックを補足する（詳細は第9章参照）。
- 補足トピックにはsource: "r7-exam"を設定
- 解説と穴埋めクイズの両方を作成

### 7.5 問題数の目安

- 1トピックあたり穴埋め問題（基礎）: 3〜8問
- 1トピックあたり穴埋め問題（応用）: 5〜15問
- 1トピックあたり4択問題: 3〜10問
- 全体で推定: 400〜700問（補足コンテンツ含む）

### 9.3 復習ノートとのギャップ（不足トピック）

### 9.4 補足コンテンツの追加方針

- 上記G-01〜G-12を新規トピックとして追加（source: "r7-exam"）
- 高優先度（G-01〜G-06）: 詳細解説 + 穴埋めクイズ + 4択クイズ
- 中優先度（G-07〜G-12）: 簡潔な解説 + 穴埋めクイズ
- UIではPDF由来コンテンツと区別して表示（「令和7年試験補足」ラベル）
